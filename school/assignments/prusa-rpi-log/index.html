Vad är det för metod jag pratar om? Jo, det finns ett flertal lösningar för att styra en 3D-skrivare med en RPI, och jag vill ha den bästa. Jag sökte efter vilka alternativ som fanns och det var dessa som jag hittade:

  • Repetier - AI baserat, förlorade redan där.
  • SimplyPrint - Kostar också

De enda valen som verkar vara bra är Klipper och Prusa link, då resterande antingen kostar, är sakta/inkompatibla eller bygger på AI, vilket jag inte kan stå för att använda. Jag läste igenom all PrusaLink dokumentation och bestämde mig till början för att använda det. För att även ha ett web-interface så krävs yttligare installationer: Fluidd är en web front-end kompatibel med Klipper men som kräver Moonraker för att kommunicera med klipper. Nu när jag kunde SSH:a mig in i RPIn så var det dags att installera Klipper. Det finns ett väldigt bra verktyg för detta som heter KIAUH (Klipper Installation And Update Helper) utvecklat av Dominik Willner (Länk till GitHub repot). Det är anpassat till linux baserade operativsystem och skapat med Raspberry Pi i fokus. Jämfört med andra installationer jag behövt gå igenom för t.ex nyare versioner av Java (21+) som inte längre stöds (natively) av RpiOS, som tagit mig några timmar att fixa och laga; så var det här väldigt väldigt lätt. Allt jag behövde göra var att först installera Git till min RPI med apt (Inbyggt i Debian, ett kommandotolksverktyg för att hantera installation, uppdatering och radering av .deb paket) vilket görs med detta kommando:

sudo apt-get update && sudo apt-get install git -y

"sudo" som prefix till ett kommando ger administrativ åtkomst och kräver ibland att du skriver ditt lösenord. apt-get är som apt, förutom att apt var designat efter användarens upplevelse och släpptes mycket senare. Skillnaden är mest bara att apt ger användaren mer info som: progress bar för installationen, färgkodat och andra saker mest för att hitta paket som kan uppgraderas. -y flaggan används för att skippa confirmation prompts som konstant frågar Y/N på det du vill installera. att vi kör två apt-get kommandon sammansatta med && är för att update även behövs utifall den senaste paketlistan inte stämmer överens med den lokala, för att undvika att "git" inte hittas. Sedan körde jag, för att installera KIAUH:

cd ~ && git clone https://github.com/dw-0/kiauh.git

som klonar (git clone [LÄNK]) KIAUH till mitt hem-directory (cd ~). Det var nu redan installerat och kunde startas via:

cd ~ && ./kiauh/kiauh.sh

Den första delen av kommandot är inte nödvändigt om du redan befinner dig i ditt hem-directory, så jag använde inte det, men jag tänkte att det vore bäst att skriva en generell lösning som tillåter att man kanske är i en annan mapp för nogrannhetens skull. Därefter, i KIAUH valde jag "install" (Navigationen är ganska self-explanatory, man skriver bara in den karaktär som motsvarar det alternativ man vill ha, och klickar enter), och sedan Klipper. Den frågar hur många instanser av klipper som ska installeras (vilket gör denna lösning bäst, då PrusaLink inte stöttar något sådant), som jag svarade med 1, för nu iallafall. Om lärarna tycker att det blir bra så hade det gått att expandera till ungefär 4 instanser av klipper (Det är det som 4GB RAM räcker till, uppskattar jag baserat på den RAM användning som jag just nu noterat). Den frågade sedan om jag ville ha ett exempel på en printer.cfg fil, vilket jag svarade ja på (Min första gång som jag jobbat med detta program, och jag lärde mig genom användning + dokumentation, länk till dokumentationen här), även fast jag ändrade typ allt i den senare. När Klipper senare installerats så fortsatte jag med Moonraker, där processen var likadan (minus antalet instanser) och till sist Fluidd som jag uppgav skulle hostas på default porten (port 80). Det var alla paket som jag behövde, men jag vill ändp poängtera att det finns även support för Touchscreen GUI (KlipperScreen) och ett paket som hanterar webcam stream för att monitorera sin print från avstånd (Crowsnest).


Man behöver också flasha 3D-skrivarens kretskort med Klipper firmware (oroa dig inte, det är såklart reversible utifall man inte längre vill använda klipper). Det kan man också göra med KIAUH, men jag valde att göra det utan den, för att ha mer kontroll då det kan bli fel om klipper t.ex har fel mikro-kontroller arkitektur så kan den inte kommunicera med skrivarens kort då de använder olika instruktionsset. För att konfigurea dessa inställningar så måste man göra en menuconfig. Man gör det genom att först gå in i klipper directoryt: cd ~/klipper och sedan make menuconfig vilket tog mig till Klipper Firmware Configuration skärmen. Prusa i3 mk3 har ett Einsy RAMBo bräda med atmega2560 arkitektur, Einsy RAMBo är då vad Prusa i3 mk3 logik bräda kallas, den använder ett AVR microchipp, därmed atmega2560 som instruktionsset. Jag valde därefter att den ska flasha via USB, vilket behöver specifieras då det också är möjligt att flasha via GPIO (Generella I/O portar). Sen escapade jag och sparade. Ursprungligen använde jag KIAUH för det, men det blev fel på just det steget, alltså att den körde på fel arkitektur. Det gjorde så att flashen failade utan att uppge detaljer, då KIAUH inte hjälper särskilt mycket just där. Det var inte särskilt problematiskt att fixa då jag bara kunde göra den manuella metoden. Iallafall nu när jag konifiguerat klipper firmware så kan jag flasha kortet genom att först stänga av klipper service, då det startas automatiskt efter installation (Det är by standard på auto-start vid start av RPIn, så man behöver inte fippla med det om man behöver flytta den eller ta ut sladden som ger den ström). För att stänga av det skrev jag sudo service klipper stop, allt information finns då beskrivet i klippers dokumentation länkat ovan (i en tidigare paragraf). Sen flashar man kortet med make flash FLASH_DEVICE=[Här stoppar man serial IDt av USB kopplingen till skrivaren]. för att få tag på IDt för USB porten så kan man köra kommandot ls /dev/serial/by-id/*; kommandot listar alla USB kopplingars IDn, men i mitt fall så var Prusan den enda som användes. "ls" kommandot är ett väldigt grundläggande linux (debian, men inte exklusivt, då det används av typ alla distros) kommando som printar en lista av alla filer / mappar i en specifierad directory. RpiOS lagrar allt detta under dev/serial, där det även finns by-path, som är väldigt likt by-id, förutom att by ID är unikt till den kopplade enheten. Om det inte funkar (det funkade ändå direkt för mig) så kan man testa att använda by-path, då det varit ett känt problem som Klipper haft problem med det i tidigare versioner. När man kör det kommandot iallafall (make flash, alltså) så flashas 3D-skrivaren Einsy RAMBo kort med klipper firmware istället för vad som den då redan använder. Om allting funkar korrekt så kommer du även se i3ns skärm lysa upp och säga att den genomgår en "firmware update" (Jag menar alltså att det var det jag såg). Alla delar av klipper är nu installerade, så då gick det helt enkelt att öppna en webbläsare, för att sedan skriva in http://teprinter, jag använde först dens IP för att komma åt web-interfacet, men insåg att det faktiskt är bättre om man använder hostname, då det inte ändras.

Steg 5: Konfigruera Klipper

Nu är jag då på sista* steget (* sista där jag ändrar på saker, att testa eller diskutera hade väl också kunnat räknas som ett steg i uppgiften): att fixa konfiguera klipper som jag just installerat. Det mesta sköter man i "printer.cfg" i ~/klipper, där kan man sätta inställningar som: serial adress (USB) till den. det skriver man så här: [mcu]
serial: /dev/by-id/[id, kan hämtas via metod nämnd tidigare (ls /dev/serial/by-id/*)]. mcu står för micro-controller, och inställningen används för att låta klipper veta hur den hittar skrivarens kort (micro-controller). resterande inställningar i printer.cfg handlar mest om saker som maxacceleration, hastighet osv. på ditt toolhead för 3D-skrivaren. Jag tog där en template anpassad till Prusa i3 mk3 som fanns här (länk). Det är generic-einsy-rambo.cfg som är gjord efter Prusa i3 mk3 (Jag nämnde Einsy RAMBo tidigare, där jag förklarade att det var vad i3:ans bräda kallades, den användes exklusivt för i3 mk3 serien). Jag har inte kollat in särskilt mycket i vilka parametrar som bör ändras, då det skulle kräva assistans av en lärare eftersom jag inte skulle röra något där.
