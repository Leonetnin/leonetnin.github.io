Info

  • OSI-modellen är en teoretisk modell som beskriver olika delar av nätverkstegnologin, olika fysiska enheter tar hand om olika delar i OSI-modellen: Router, Switch och Kabel.
  • De sju lagrerna i OSI-modellen:
    1. Applikationslagret - Användare använder det för att kommunicera med program (applikationer)
    2. Presentationslagret - Ett lager som hanterar dataöversättning, kryptering och komprimering.
    3. Sessionslagret - Öppnar och upprätthåller sessioner mellan klienter.
      - Simplex: envägskommunikation;
      - Halfduplex: Tvåvägskommunikation men bara en i taget;
      - Full-duplex: Tvåvägskommunikation
    4. Transportlagret - Används för att t.ex bestämma vad som händer om något går fel i transporten. TCP och UDP är två olika sätt som internetkommunikation kan transporteras med. TCP tar längre tid men ser till att allt kommer fram korrekt. UDP går snabbare men data kan förloras eller komma i fel ordning, används oftast för att strömma video.
    5. Nätverkslagret - Ett lager som hittar rätt väg genom nätverket för data att skickas. Routrar.
    6. Datalänklagret - Lagret som sköter kommunikation mellan fysiska enheter.
    7. Fysiska lagret - Kablarna som skickar datat, som t.ex en ethernet-kablar.
  • TCP/IP är en arkitektur för datakommunikation, lik OSI-modellen förutom med 4-5 lager (4 som vi använder på DNTn)
  • De fyra lagrerna i TCP/IP modellen:
    1. Applikationslagret - Slår ihop de tre högsta lagren i OSI-modellen (Applikation, Presentation, Session). HTTP(HyperText Transfer Protocol) - hemsidor på internet, HTTPS - krypterad version av HTTP, FTP - File transfer protocol skickar filer mellan server och klient. DHCP - Tilldelar IP-adresser automatiskt. DNS - Översätter domännamn till IP-adresser.
    2. Transportlagret - TCP och UDP (motsvarar OSIs transportlager)
    3. Nätverkslagret - IPv4 och IPv6 (motsvarar OSIs nätverkslager)
    4. Datalänklagret - Slår ihop OSI-modellens fysiska och datalänk lager, Hanterar MAC adresser: datorns fysiska adress. Kommunikation med t.ex wi-fi eller ethernetsladd som undviker kollisioner som sker i överföringen.
  • Står för Network adress translation, Tekniken gör det möjligt att ansluta många datorer till en internetanslutning med en eller några gemensamma IP-adresser. När en dator ska hämta en hemsida så översätter routern datorns privata IP-adress till routerns offentliga IP-adress. Routern håller koll på vilken dator som just skickade ett meddelande så den kan skicka svaret till rätt privata IP-adress genom att åter översätta den. (det finns inte nog med IPv4 adresser för varje enhet)
  • Varje router (och även datorer) har flera portar. På routern konifigurerar man en regel som kopplar ihop en port med en privat IP-adress. När man sedan frågar att komma åt servern måste man alltså både veta routerns offentliga IP och vilken port man vill komma åt.
